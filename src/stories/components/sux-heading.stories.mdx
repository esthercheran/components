# Typography / Heading

Heading is a typography component used to create various levels of hierarchies between text.

export const DefaultHeading = args => {
  return html`
    <div style="padding: 10%; justify-content: center; gap:16px;">
      <sux-heading .size="${args.size}" classification="serif" .weight="${args.weight}"> Headings are great to create hierarchy. </sux-heading>
      <br />
      <sux-heading .size="${args.size}" classification="sans-serif" .weight="${args.weight}"> Headings are great to create hierarchy. </sux-heading>
      <br />
      <sux-heading .size="${args.size}" classification="mono" .weight="${args.weight}"> Headings are great to create hierarchy. </sux-heading>
    </div>
  `;
};

<Canvas>
  <Story name="Playground">{DefaultHeading.bind()}</Story>
</Canvas>

## Table of options / API

From the design point of view, each component has a number of options. These options and their names are platform agnostic, and each implementation should adapt these to fit into their framework.

### Size

Headings have a variety of sizes available. These sizes are defined using T-shirt sizing in order to support multiple scales without direct correlation to pixel values.

export const HeadingSize = args => {
  return html`
    <div style="padding: 10%; justify-content: center; gap:16px;">
      <sux-heading size="xs" classification="sans-serif" .weight="${args.weight}"> Extra Small heading </sux-heading>
      <sux-heading size="s" classification="sans-serif" .weight="${args.weight}"> Small heading </sux-heading>
      <sux-heading size="m" classification="sans-serif" .weight="${args.weight}"> Medium heading </sux-heading>
      <sux-heading size="l" classification="sans-serif" .weight="${args.weight}"> Large heading </sux-heading>
      <sux-heading size="xl" classification="sans-serif" .weight="${args.weight}"> X Large heading </sux-heading>
      <sux-heading size="xxl" classification="sans-serif" .weight="${args.weight}"> XX Large heading </sux-heading>
      <br />
    </div>
  `;
};
