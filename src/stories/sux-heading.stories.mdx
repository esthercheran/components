# Heading

Heading is a typography component used to create various levels of hierarchies between text.

export const DefaultHeading = (args) => {
    return html`
        <div style="padding: 10%; justify-content: center; gap:16px;">
            <sux-heading 
                .size="${args.size}"
                .variant="${args.variant}"
                .weight="${args.weight}"
                .tag="${args.tag}"
            >
                Headings are great to create hierarchy.
            </sux-heading>
            <br/>
            <sux-heading 
                size="l"
                variant="sans-serif"
                weight="regular"
            >
                Headings are great to create hierarchy.
            </sux-heading>
            <br />
            <sux-heading 
                size="l"
                variant="mono"
                weight="regular"
            >
                Headings are great to create hierarchy.
            </sux-heading>
        </div>
    `
}

## Table of options / API
From the design point of view, each component has a number of options. These options and their names are platform agnostic, and each implementation should adapt these to fit into their framework.

### Size
Headings have a variety of sizes available. These sizes are defined using T-shirt sizing in order to support multiple scales without direct correlation to pixel values.
